# L1 — High-Level Requirements

## Raffle Web Application

---

### L1-001: Public Raffle Drawing

The system shall provide a public-facing web page that allows visitors to draw a name from the currently active raffle. Once a name has been drawn, it shall not be drawn again for that raffle. The draw experience shall include an engaging animation that cycles through names before revealing the winner, followed by a celebratory visual effect.

---

### L1-002: Unique Draw Enforcement

The system shall guarantee that each name in a raffle can only be drawn once. Drawn names shall be recorded and excluded from all subsequent draws until the raffle is reset by an administrator.

---

### L1-003: Back Office Administration

The system shall provide a back office web application where authenticated administrators can create, configure, and manage multiple raffles, each with its own distinct list of names. Administrators shall be able to customize the public-facing raffle page per raffle, including a heading, subheading, and selection of visual theme and animation style.

---

### L1-004: Raffle Activation

The system shall allow an administrator to set exactly one raffle as active at any given time. Only the active raffle shall be presented on the public-facing drawing page.

---

### L1-005: Authentication and Authorization

The back office application shall require user authentication. Only authenticated users with administrator privileges shall be able to access back office functionality.

---

### L1-011: Draw Animation and Visual Experience

The public-facing raffle page shall present an animated draw experience that builds anticipation. Upon triggering a draw, the interface shall rapidly cycle through candidate names with an accompanying sound effect, gradually decelerate, and come to rest on the selected winner. A celebratory animation shall play upon reveal. The animation style, visual theme, and sound shall be configurable per raffle by an administrator.

---

### L1-006: Responsive Design

Both the public-facing raffle page and the back office application shall be fully responsive, providing an optimal user experience across all viewport sizes from extra-small (xs) to extra-large (xl) screens.

---

### L1-007: Security

The system shall be built following industry-standard security best practices, including protection against OWASP Top 10 vulnerabilities, secure session management, and encrypted data transmission.

---

### L1-008: Accessibility

The system shall conform to WCAG 2.2 Level AA accessibility guidelines, ensuring all functionality is usable by people with disabilities, including those relying on assistive technologies.

---

### L1-009: Code Quality and Maintainability

The system shall be developed using modern best practices for code quality, including consistent coding standards, automated testing, modular architecture, and comprehensive documentation to support long-term maintainability.

---

### L1-010: Reliability

The system shall ensure data integrity and consistent behavior under concurrent usage. Draw results shall be durable and the system shall handle error conditions gracefully without data loss.
